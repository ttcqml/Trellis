event-builder 的 target_node 规则不够清晰
 问题: skill 文档中 target_node 规则写的是：
System → "System"
Entity → "." 或子节点

但实际使用中，set_node_enabled 的 target_node 应该是 场景中的节点名（如 "FMonsterTest"），而不是 "."。文档没有明确说明当动作要操作场景中其他节点时，target_node 应该填什么。
 优化建议: 补充说明：
"." = 事件表所属节点自身
"NodeName" = 场景根节点下的同级节点（如 "FMonsterTest", "Door"）
"System" = 全局系统节点

 优化建议: 在 skill 文档的脚本模板中，默认加上编码设置： import sys sys.stdout.reconfigure(encoding='utf-8', errors='replace')
